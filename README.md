# Byreal 델타 헤지 파밍 봇

Byreal DEX(Bybit 인큐베이팅, Solana)에서 **CLMM LP + Perps Short 델타 헤지** 전략으로 방향 무관 수수료 수익을 자동화하는 봇입니다.

## 전략 개요

```
┌─────────────────────────────────────────────────────────┐
│  목표: ETH 가격 방향과 무관하게 수수료 + 펀딩레이트 수익  │
└─────────────────────────────────────────────────────────┘

  [자본 70%] CLMM LP 예치 (ETH/USDC, ±20% 범위)
       → 거래 수수료 수익 발생

  [자본 30%] Hyperliquid Perps Short (LP 델타만큼)
       → LP의 ETH 가격 노출 상쇄 + 펀딩레이트 수령

  ETH 가격 변동 → LP 델타 변화 → 주기적으로 Short 규모 조정
```

### 수익원

| 항목 | 설명 | 방향 의존성 |
|------|------|------------|
| LP 수수료 | 거래량 × 수수료율 × 내 비중 | 없음 |
| 펀딩레이트 | Short 포지션 보유 시 수령 (bullish 시장) | 낮음 |
| Perp PnL | 헤지 용도 (수익 목적 아님) | 있음 (IL 상쇄) |

## 설치

```bash
# 1. 의존성 설치
pip install -r requirements.txt

# 2. 환경변수 설정
cp .env.example .env
# .env 파일을 편집해서 설정값 조정
```

## 실행

```bash
# 페이퍼 트레이딩 (기본값)
python main.py
```

종료: `Ctrl+C` → 최종 손익 리포트 출력 후 종료

---

## 설정 (.env)

| 변수 | 기본값 | 설명 |
|------|--------|------|
| `INITIAL_CAPITAL` | `1000` | 초기 자본 (USDC) |
| `LP_TOKEN` | `ETH` | LP 대상 토큰 |
| `LP_FEE_TIER` | `0.3` | 수수료율 % |
| `LP_RANGE_PCT` | `20` | LP 범위 ±% |
| `LP_CAPITAL_RATIO` | `0.7` | LP 투입 비율 |
| `REBALANCE_INTERVAL` | `300` | 업데이트 주기 (초) |
| `DELTA_THRESHOLD` | `0.05` | 리밸런싱 발동 델타 편차 |
| `ESTIMATED_POOL_DAILY_VOLUME` | `7500000` | 풀 일 거래량 추정 (수수료 계산용) |
| `ESTIMATED_POOL_TVL` | `3000000` | 풀 TVL 추정 |
| `LOG_INTERVAL` | `60` | 리포트 주기 (초) |

## 데이터 출처

- **가격 / 펀딩레이트**: [Hyperliquid 공개 API](https://api.hyperliquid.xyz/info) (인증 불필요)
- **LP 수수료 시뮬레이션**: Byreal 추정 거래량 기반 계산
- **CLMM 수학**: Uniswap V3 공식 적용

## 주의사항

- 현재는 **페이퍼 트레이딩 전용**입니다. 실거래 연동 미구현.
- LP 수수료는 추정치이며 실제 풀 거래량에 따라 달라집니다.
- 펀딩레이트는 시장 상황에 따라 역전될 수 있습니다 (Short이 비용을 지불).
- 범위 이탈 시 수수료 수익이 0이 되므로 `LP_RANGE_PCT` 조정이 중요합니다.
